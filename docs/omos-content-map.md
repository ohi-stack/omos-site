## Operational Rules

1. Every public page must map to a documented implementation path.
2. Marketing claims must not exceed implemented functionality.
3. Runtime manifests should expose only production-safe metadata.
4. Governance language must remain separated from commercial LLC product positioning.
5. Every major feature should have:
   - documentation
   - implementation notes
   - route mapping
   - production checklist

## Deployment Goal

OMOS.OneGodian.com should function as:
- a public explanation layer
- a runtime documentation node
- a developer-facing protocol site
- a product and download gateway
- an AI-era identity and orchestration framework
